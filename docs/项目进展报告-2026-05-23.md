## 一、已完成成果

### 1.1 前端（client/）— 完整可运行

所有 UI 组件已开发完毕，Demo 模式下可独立运行，无需后端。

| 组件 | 功能 |
|------|------|
| `SearchBox.jsx` | 搜索输入框，支持 Ctrl+Enter 提交、示例问题快捷填入、加载中防重复提交 |
| `ResultCard.jsx` | 结构化排故结果展示：故障概述 / 可能原因（概率标签）/ 处理步骤 / AMM 章节 / 工具航材。每张卡片可独立折叠展开 |

| `MenuCard.jsx` | 侧边栏可展开卡片（历史记录 / 设置） |
| `AvatarDropdown.jsx` | 右上角头像下拉面板，显示账户信息和 API 状态 |
| `SidebarSection.jsx` | 侧边栏三级折叠菜单组件 |
| `useHistory.js` | localStorage 历史记录管理 Hook（增删查清，上限 50 条） |
| `api.js` | API 调用封装：优先调后端，后端不可达时自动降级为本地 Mock |

**启动方式**：
```bash
cd client
npm install
npm run dev        # → http://localhost:5173
```

## 五、已知问题 & 下一步

### 当前阻塞点：Embedding（文本向量化）

中转站不支持 embedding 模型，需要替代方案：

| 方案 | 优点 | 缺点 |
|------|------|------|
| HuggingFace 云端推理 API | 免费、无需下载 | 需注册 HF 账号 |
| 本地 Transformers.js | 完全离线 | Windows 兼容性差，内存占用大 |
| 换中转站 | 不改代码 | 需找支持 embedding 的中转 |

**推荐方案 A**：注册 HuggingFace 账号 → 获取 Token → 修改 retriever.js 调用云端 embedding → 完整 RAG 跑通。

### 当前阻塞点：知识库导入脚本 OOM

import-kb.mjs 在 bash 终端中偶发内存溢出。换纯净终端（PowerShell 或新开 Git Bash）即可正常执行。
