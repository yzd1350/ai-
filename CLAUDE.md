# CLAUDE.md — Claude Code 项目规则

## 1. 你是谁 / 项目背景

你正在帮助一位**飞机机电设备维修专业的大学生**开发"机务AI排故助手"。他是**编程初学者**，你的代码和沟通要适应这一点。

项目完整规格见：[docs/prd/机务AI排故助手-PRD.md](docs/prd/机务AI排故助手-PRD.md)
AI 通用说明见：[AGENTS.md](AGENTS.md)

## 2. 沟通规范

- **解释用大白话**：避免术语堆砌，用比喻和类比帮助理解
- **改动后一句话总结**：每次代码改动后，用 1-2 句话说明改了什么、为什么
- **不要过度解释**：简单改动不需要长篇说明；复杂概念主动简化
- **遇到问题先排查**：别猜，用工具验证（读文件、看编译输出、curl 测试）

## 3. 代码规范

- **注释用中文**，写清楚 WHY 而不是 WHAT
- **默认不写注释**，只在非常规逻辑处加一行短注释
- **别过度设计**：三个相似行不急着抽象，一个函数够了就别拆
- **不引入新依赖**，除非确有必要（先问自己能不能用已有代码解决）
- **不写半成品代码**：每个功能要么完整可运行，要么不做
- **别加防御性代码**：信任内部调用和框架保证，只在系统边界校验

## 4. 技术约束

| 规则 | 说明 |
|------|------|
| 前端框架 | React 19 + Vite + Tailwind CSS 4，不准换 |
| 后端框架 | Express + LangChain.js，不准换 |
| 大模型 | DeepSeek V4 Pro（OpenAI 兼容接口） |
| 向量化 | OpenAI text-embedding-3-small（或中转站） |
| 加载顺序 | 不要安装新包除非确有必要 |
| 移动端优先 | 所有 UI 改动先考虑 375px 宽度效果 |

## 5. 项目特定约束

- **单用户模式**：不做注册、登录、权限
- **localStorage 存历史**：上限 50 条，不做服务端历史
- **API Key 只放后端 .env**：前端用 fetch 调后端，不直接调 AI
- **Mock 数据保留**：`api.js` 的 mock 模式在正式上线前不要删，方便前后端独立开发
- **知识库为空是合法状态**：后端要给出友好提示，不要 crash

## 6. 开发流程

1. **读 PRD** → 确认需求不偏离
2. **读相关代码** → 了解现有实现
3. **改代码** → 最小改动，一次只改一件事
4. **验证** → `npm run dev` 启动，浏览器看效果
5. **总结** → 告诉用户改了什么
